2. buatlah contoh flowchart rumus bangun datar ​


jawab

berikut flowchart utk rumus bangun datar menggunakan java:

start

input jenis bangun datar

input nilai sisi, panjang, dan lebar (jika diperlukan)

input nilai jari-jari atau diameter (jika diperlukan)

if jenis bangun datar = "persegi" then

   hitung luas = sisi * sisi

   hitung keliling = 4 * sisi

   output luas dan keliling

else if jenis bangun datar = "persegi panjang" then

   hitung luas = panjang * lebar

   hitung keliling = 2 * (panjang + lebar)

   output luas dan keliling

else if jenis bangun datar = "lingkaran" then

   hitung luas = phi * jari-jari * jari-jari

   hitung keliling = 2 * phi * jari-jari

   output luas dan keliling

else if jenis bangun datar = "segitiga" then

   hitung luas = 0.5 * alas * tinggi

   hitung keliling = sisi1 + sisi2 + sisi3

   output luas dan keliling

end if

end

8. jelaskan definisi flowchart dan contoh jenisnya​


Jawaban:

Pengertian Flowchart ( Bagan Alir ) adalah bagan  (chart) yang menunjukkan alir  (flow) di dalam program atau prosedur sistem secara logika. Bagan alir (flowchart) digunakan terutama untuk alat bantu komunikasi dan untuk dokumentasi.

jenis jenisnya

Bagan alir sistem (systems flowchart).

Bagan alir dokumen (document flowchart).

Bagan alir skematik (schematic flowchart).

Bagan alir program (program flowchart).

Bagan alir proses (process flowchart).

9. Untuk menuliskan sebuah rumus di flowchart


Jawaban:

menuliskan sebuah rumus di flowchart:

Untuk pengolahan data dengan komputer dapat dirangkum urutan dasar untuk pemecahan suatu masalah yaitu;

START : Berisi intruksi untuk persiapan peralatan yang di perlukan sebelum menangani pemecahan masalah

READ : Berisi instruksi untuk membaca data dari suatu peralatan input.

PROCESS : Berisi kegiatan yang berkaitan dengan pemecahan persoalan sesuai dengan data yang dibaca.

WRITE  : Berisi instruksi untuk merekam hasil kegiatan ke perlatan output.

END : Mengakhiri kegiatan pengolahan

Bila seorang analis dan programmer akan membuat flowchart, ada beberapa petunjuk yang harus diperhatikan seperti:

Flowchart digambarkan dari halaman atas ke bawah dan dari kiri kekanan.

Aktivitas yang digambarkan harus didefinisikan secara hati-hati dan definisi ini harus dapat di mengerti oleh pembacanya

Kapan aktivitas dimulai dan berakhir harus ditentukan secara jelas.

Setiap langkah dari aktivitas harus diuraikan dengan menggunakan deskripsi kata kerja

Setiap langkah dari aktivitas harus berada pada urutan yang benar.

Lingkup dan range dari aktifitas yang sedang digambarkan harusditelusuri dengan hati-hati. Percabangan-percabangan yang memotong aktivitas yang sedang digambarkan tidak perlu digambarkan pada flowchart yang sama. Simbol konektor harus digunakan dan percabangannya diletakan pada halaman yang terpisah atau hilangkan seluruhnya bila percabangannya tidak berkaitan dengan sistem.

Gunakan simbol-simbol flowchart yang standar.

Penjelasan:

Flowchart terbagi atas 5 jenis, yaitu :

Bagan alir sistem (systems flowchart).

Bagan alir dokumen (document flowchart).

Bagan alir skematik (schematic flowchart).

Bagan alir program (program flowchart).

Bagan alir proses (process flowchart).

25. Tolong buatkan flowchart untuk algoritma ini pls... ini aldoritma induksi matematika


Jawaban:

Lihat yang flow chart ijo ya,

ganti yang bagian dalamnya jadi gini :

Start : R = 1, K = m

Condition : K > 0

Do Task : R = R *  P; K = K - 1

Itu kan buat ngitung P kalo dipangkatin M kan, jadi yang bikin program maksudnya gini :

1. R sebagai variable sementara menyimpan hasil perkalian

2. R = 1 itu sebagai awal iterasi

3. K = m berarti menentukan nanti berapa kali P itu harus dipangkatkan (yaitu sebanyak m), jadi K < 0, K = K - 1 itu untuk memastikan perkaliannya diulangi sebanyak m kali.

Alternative iterasinya bisa K = 0 di awal, kemudian selama K < m maka K = K + 1

4. Tiap iterasi, R  akan menyipan hasil perkalian sebelumnya terus dikalikan lagi dengan dirinya sendiri (konsep perpangkatan)

jadi misalnya P pangkat M nanti iterasi nya kayak gini :

iterasi 1 : 1 * P

iterasi 2 : 1 * P * P

iterasi 3 : 1 * P * P * P

iterasi m : 1 * P * P * P * ..... * Pm
